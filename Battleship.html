Project Overview

Game Summary: Battleships is a Python terminal game where players strategically place ships before taking turns attempting to sink the opponent's fleet. The game features randomized computer ship placement, player-controlled ship placement, turn-based gameplay, and win/loss conditions.

Core Goal: The project was designed to recreate the classic Battleships board game while strengthening my understanding of object-oriented programming, game state management, user input validation, and algorithmic thinking.

Development Context: Battleships was developed as a solo Python project and deployed on Heroku. The project focused on building complete gameplay systems using Python while following clean coding practices and modular design.

What I Learned: Throughout development I gained experience working with object-oriented programming, board management, player input validation, game loops, collision detection for ship placement, debugging, and Python best practices.

Biggest Challenge: One of the biggest challenges was creating a ship placement system that correctly handled overlapping ships, board boundaries, and both horizontal and vertical placement while preventing invalid user input.

Gameplay Features

Ship Placement and Board Management

Players strategically position ships of varying sizes before the match begins. Ships can be placed horizontally or vertically while the game validates every placement to prevent overlapping ships or positions outside the board.

Implemented Features

  • Player ship placement.
  • Random computer ship placement.
  • Horizontal and vertical placement.
  • Multiple ship sizes.
  • Placement validation.
  • Overlap detection.

Turn-Based Combat

Players and the computer take turns attempting to locate and destroy each other's fleet. Hits and misses are tracked throughout the game until one player's ships have all been destroyed.

Gameplay Systems

  • Alternating player turns.
  • Hit detection.
  • Miss detection.
  • Board state updates.
  • Win condition checking.
  • Game over system.

Game And Logic Validation

The game uses object-oriented programming to manage multiple boards simultaneously. Extensive input validation ensures players cannot place ships illegally or enter invalid coordinates during gameplay.

Gameplay Systems

  • Board class architecture.
  • Multiple board management.
  • User input validation.
  • Coordinate checking.
  • Boundary validation.
  • Game state management.

Technical Notes

Battleships was built using Python following an object-oriented architecture centred around a reusable Board class. Multiple board instances manage both player and computer game states, while validation systems ensure legal ship placement and accurate gameplay throughout each match.
